## Read First
- `01_context/voice-and-style.md` — non-negotiable. Every draft must sound like the user.
- `03_people/` — check the relevant person's page if the draft is for or about a specific person

## Process

1. If you don't know audience, format, or length — ask one question. Not multiple.
2. Draft immediately. No outlining first.
3. Save to `02_projects/[descriptive-name].md` with `Status: Draft`
4. Silently check against voice-and-style.md before presenting:
   - No anti-patterns (buzzwords, filler, hedging)
   - Starts with the insight, not context-setting
   - Right compression for the mode
   - Sounds like the user talking, not a document

## Format by Type

**Email or message:** Direct opening, one clear ask or update, max 150 words unless complexity requires more.

**Internal update:** What happened, what it means, what's next. Assumes shared context.

**Strategy or proposal:** Problem → Approach → Evidence → Ask. Headers only where they genuinely help.

**Public content (LinkedIn, etc.):** Hook in line 1 — insight, not a question. Concrete and specific. No engagement bait.

**Meeting prep:** Goal of meeting / key points to make / questions to ask / what needs to be decided.

## After Drafting
State what was drafted and where it's saved. Note one thing you weren't sure about and how you resolved it. Ask one question: "Does this hit the right tone? Or should I adjust [X]?"
